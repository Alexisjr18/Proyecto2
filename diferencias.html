DIFERENCIAS





Los datos transmitidos por el protocolo HTTP no están encriptados, es decir, texto plano. Por lo tanto, es muy inseguro usar el protocolo HTTP para transmitir información privada.
Para cifrar los datos transmitidos por el protocolo HTTP, nació HTTPS.
En términos simples, el protocolo HTTPS es un protocolo de red creado por el protocolo SSL + HTTP que puede llevar a cabo la transmisión encriptada y la autenticación de identidad, que es más segura que el protocolo http.


Las diferencias entre HTTPS y HTTP son las siguientes:


  • 1.- El protocolo https debe solicitar un certificado en ca.
    Generalmente, hay menos certificados gratuitos, por lo que se requiere una cierta tarifa.

  • 2.- Http es un protocolo de transmisión de hipertexto, la información se transmite en texto plano y https es un protocolo seguro de transmisión encriptada SSL.

  • 3. Http y https utilizan métodos de conexión completamente diferentes, con puertos diferentes:
    el primero es 80 y el último es 443.

  • 4. La conexión de http es muy simple y sin estado, el protocolo HTTPS es un protocolo de red creado por el protocolo SSL + HTTP que puede llevar a cabo la transmisión encriptada y la autenticación de identidad, que es más segura que el protocolo http.
